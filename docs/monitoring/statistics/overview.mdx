# Statistics

<Admonition type="note" title="">

* RavenDB reports runtime statistics and metrics for the server and for individual databases.  

* You can view this data in Studio, or retrieve it in JSON format from dedicated endpoints.  

* RavenDB can also be monitored via protocols like SNMP and OTLP (used by OpenTelemetry),
  as well as monitoring tools like Prometheus.  

* In this article:
   * [Server statistics](overview.mdx#server-statistics)

### Server statistics via Studio

Find server statistics in the following Studio views:

* [Cluster Dashboard](../../studio/cluster/cluster-dashboard/cluster-dashboard-overview.mdx): live per-node CPU, memory, traffic, indexing, and storage metrics.
* [Cluster View](../../studio/cluster/cluster-view.mdx#cluster-view-stats): cluster-wide stats, including assigned and licensed cores, election term, and per-node state and memory.

### Metrics

<Admonition type="note" title="">
Get server metrics using the `{serverUrl}/admin/metrics` endpoint.
</Admonition>

A document containing the following metrics is retrieved:

* **Requests**
    * RequestsPerSec
    * ConcurrentRequestsCount
    * AverageDuration
* **Docs**
    * PutsPerSec
    * BytesPutsPerSec
* **Attachments**
    * PutsPerSec
    * BytesPutsPerSec
* **Counters**
    * PutsPerSec
    * BytesPutsPerSec
* **TimeSeries**
    * PutsPerSec
    * BytesPutsPerSec
* **MapIndexes**
    * IndexedPerSec
* **MapReduceIndexes**
    * MappedPerSec
    * ReducedPerSec

The rate fields (those ending in `PerSec`) are **meters**, each with the following format.  
`ConcurrentRequestsCount` and `AverageDuration` are plain numeric values.

```json
{
    "Current": 0.0,
    "Count": 131,
    "MeanRate": 1.8,
    "OneMinuteRate": 0.2,
    "FiveMinuteRate": 0.4,
    "FifteenMinuteRate": 0.1
}
```

</ContentFrame>

### CPU stats

<Admonition type="note" title="">
Get server CPU stats using the `{serverUrl}/admin/debug/cpu/stats` endpoint.
</Admonition>

A document is retrieved with the process CPU times and the current thread-pool thread counts.

```json
{
    "@metadata": {
        "DateTime": "2026-07-20T09:44:04.9416220Z",
```
