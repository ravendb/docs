# Vector search: Start

Vector search operations let you find semantically similar content by comparing [Embeddings](https://en.wikipedia.org/wiki/Embedding_(machine_learning)).  
You can search over embeddings pre-generated by [ongoing embeddings-generation tasks](../ai-integration/generating-embeddings/embeddings-generation_start), or generate the embeddings for your documents on-the-fly during the search operation.  
When you run a vector search, your search query is converted into an embedding as well, and compared against document embeddings using either a dynamic query for quick searches or a static index for optimized performance.  
This enables semantic searches based on meaning rather than exact keyword matches.  
- Vector search can be used by other RavenDB AI features, including GenAI tasks and AI agents.  
- Vector search can create embeddings using an external AI provider (such as OpenAI) or RavenDB's default embedding model. (Be sure to use the same model to both generate and query the embeddings.)
