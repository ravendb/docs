# PostgreSQL Protocol: Overview
<Admonition type="note" title="">

* RavenDB implements the PostgreSQL protocol, allowing applications and tools that 
  use PostgreSQL, like [Power BI](../../integrations/postgresql-protocol/power-bi.mdx) and pgAdmin, 
  to retrieve data from a RavenDB database.  

* Clients can query the database in SQL, or in RavenDB's native 
  [RQL](../../querying/rql/what-is-rql.mdx) (Raven Query Language).  

* To use RavenDB as a PostgreSQL server you need -  
   * A [license](../../integrations/postgresql-protocol/overview.mdx#license) that enables the PostgreSQL Protocol.  
   * PostgreSQL explicitly enabled in your [settings](../../integrations/postgresql-protocol/overview.mdx#settings).  

* [Installing](../../start/installation/setup-wizard/overview.mdx) RavenDB as 
  a [secure](../../server/security/overview.mdx) server allows you to authenticate 
  PostgreSQL clients, granting access only to clients that provide the proper credentials.  

* The PostgreSQL protocol is not supported for sharded databases.

<Panel heading="Enabling PostgreSQL support">

<ContentFrame>

### License

* PostgreSQL is included in all licenses.  
* To [connect Power BI to RavenDB](../../integrations/postgresql-protocol/power-bi.mdx), 
  your license must explicitly enable Power BI.  
  If your current license doesn't include Power BI Support, you can acquire one that does [here](https://ravendb.net/buy).  
* Visit Studio's [About](../../licensing/overview.mdx#studio-license-management) 
  page to see the features included in your license.  

</ContentFrame>

<ContentFrame>

### Settings

* PostgreSQL support is an [experimental feature](../../server/configuration/core-configuration.mdx#featuresavailability) 
  and must be explicitly enabled in your [settings](../../server/configuration/configuration-options.mdx#settingsjson).  
  To enable the feature, add these two lines to your server's `settings.json` file and 
  restart the server:  

  ```json
  "Integrations.PostgreSQL.Enabled": true,
  "Features.Availability": "Experimental"
  ```

</ContentFrame>

<ContentFrame>

### PostgreSQL Port

* To access RavenDB, your clients need not only the server's **URL** but also the 
  PostgreSQL **Port** number.  
  By default, the port number is *5433*.  
* To use a different port, add the following line to your `settings.json` file, with a port number 
  of your choice, and restart the server:  

  ```json
  "Integrations.PostgreSQL.Port": 5433
  ```

</ContentFrame>

<ContentFrame>

### Platform support

* To parse incoming SQL, the PostgreSQL integration uses `libpg_query`, a library that is 
  built separately for each platform (like Windows x64 or Linux ARM64) and ships with 
  the RavenDB server.  
* When the RavenDB server runs on a platform that this library does not ship for, like the 
  Raspberry Pi (32-bit ARM) package, the PostgreSQL integration will not start even if 
  enabled in your settings, and the server will log a warning.  

</ContentFrame>

</Panel>

<Panel heading="Supported clients">

RavenDB supports these PostgreSQL clients:

| Client | Description |
|--------|-------------|
| **[Power BI](../../integrations/postgresql-protocol/power-bi.mdx)** (the Power BI Desktop application and the online service) | Build reports and dashboards over RavenDB data. |
| **[pgAdmin](https://www.pgadmin.org/)** | Browse a RavenDB database and run SQL or RQL queries using the Query Tool. |
| **[Microsoft Fabric](https://learn.microsoft.com/en-us/fabric/fundamentals/microsoft-fabric-overview)** | Copy data out of a RavenDB database using a Fabric [Copy job](https://learn.microsoft.com/en-us/fabric/data-factory/what-is-copy-job). |
| **[Npgsql](https://www.npgsql.org/)** | The .NET data provider for PostgreSQL. Versions 6 through 10 are supported. |

</Panel>

<Panel heading="Querying over the protocol">

Clients that connect to a RavenDB database over PostgreSQL see each document collection 
as a table in a `public` schema.  
You can query in two languages:  

* **SQL**  
  The RavenDB server translates SQL `SELECT` queries to RQL and runs the translated 
  queries on the database.  
  See the supported SQL subset and its limitations in 
  [Querying with SQL](../../integrations/postgresql-protocol/querying-with-sql.mdx).  

* **RQL**  
  RavenDB recognizes RQL input and runs the query as it is.  
  e.g., `from Orders where ShipTo.Country == 'France'`.  

</Panel>

<Panel heading="Security">

To allow RavenDB to authenticate PostgreSQL clients:  

* Set RavenDB as a [Secure Server](../../server/security/overview.mdx).  
  <Admonition type="warning" title="">
  If RavenDB is not set as a secure server, the server will require no authentication over the PostgreSQL protocol.  
  Allowing just any client to connect to your database (via PostgreSQL or otherwise) without authentication 
  is risky, and should in general be avoided.  
  </Admonition>

* Create [PostgreSQL Credentials](../../studio/database/settings/integrations.mdx) using Studio.  
  PostgreSQL credentials are a **user name** and a **password** that a client must provide 
  to access the database.  

<Admonition type="note" title="">
A secure server accepts PostgreSQL connections only over TLS.  
The server refuses a non-TLS connection before the client sends its password, so the 
password never travels unencrypted.  
To connect to a secure server, enable SSL in your client's connection settings.  
e.g., in an Npgsql connection string, include `SSL Mode=Require`.  
</Admonition>

</Panel>
