<Admonition type="note" title="">

* Queries can include a `where` clause to **filter the results** returned from a collection or an index.
    
* Filtering is performed on the server: the `where` clause is sent as part of the query, evaluated against an index (static or auto), 
  and only documents that match the specified condition are returned to the client.   

  For example, when querying a collection of products, you can filter the results to return only products that are in stock, or only products whose price is above a specified value.
    
* You can filter by simple field values, numeric ranges, nested fields, values inside collections, document IDs, and more.
    
* The examples in this article mostly demonstrate **dynamic queries** (queries that don't specify which index to query).  
  You can use the same filtering syntax when querying a **static index**,
  as explained below in [Filtering when querying static indexes](#filtering-when-querying-static-indexes).

<Panel heading="Filtering and index usage">
    
* **When querying a collection with a filtering condition without specifying an index**,  
  RavenDB treats the query as a dynamic query and will use an **AUTO-index** to process the filter.
    
  If no suitable auto-index exists yet, RavenDB will create one automatically based on the fields used in the query.
  Later queries that need the same indexed fields will reuse the same auto-index.  
  To learn more about dynamic queries and auto-indexes, see [Query overview](../../../querying/overview.mdx#2-query-a-collection---with-filtering).    
    
* **When querying a STATIC index**, RavenDB will use the specified index to process the filter.  
  The filtering condition is evaluated against the fields indexed by that index, and no auto-index is created. 
    
</Panel>

<Panel heading="Filter by field value">
    
Use `whereEquals` to return only documents where a field has the specified value.  
For example, the following query will return all _Employee_ documents where `FirstName` is `"Robert"`.    
    
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const employees = await session
    .query({ collection: "Employees" })   // Query the Employees collection
    .whereEquals("FirstName", "Robert")   // Apply the filter predicate
    .all();                               // Execute the query and return the results
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const employees = await session.advanced
    .rawQuery("from Employees where FirstName = $firstName")
    .addParameter("firstName", "Robert")
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Employees
where FirstName = "Robert"
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/Employees/ByFirstName`  
> The auto-index is derived from the `FirstName` field referenced in the `where` clause.  
    
</Panel>   

<Panel heading="Filter by numeric value">

You can filter numeric fields using comparison operators.  
For example, the following query returns all products that have more than `50` units in stock.   
    
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const products = await session
    .query({ collection: "Products" })
    .whereGreaterThan("UnitsInStock", 50)
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const products = await session.advanced
    .rawQuery("from Products where UnitsInStock > $units")
    .addParameter("units", 50)
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Products
where UnitsInStock > 50
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/Products/ByUnitsInStock`  
> The auto-index is derived from the `UnitsInStock` field referenced in the `where` clause.  
>    
> The auto-index name does not depend on the comparison operator used in the filtering condition,  
> so the same auto-index serves any comparison on `UnitsInStock`.
    
You can use the following comparison methods:

| Method                    | Description              |
| ------------------------- | ------------------------ |
| `whereEquals`             | Equal to                 |
| `whereNotEquals`          | Not equal to             |
| `whereGreaterThan`        | Greater than             |
| `whereGreaterThanOrEqual` | Greater than or equal to |
| `whereLessThan`           | Less than                |
| `whereLessThanOrEqual`    | Less than or equal to    |

</Panel>

<Panel heading="Filter by nested field">
    
To filter by a nested field, specify the full path to the field.  
For example, the following query returns all orders that were shipped to `Albuquerque`.    
    
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const orders = await session
    .query({ collection: "Orders" })
    .whereEquals("ShipTo.City", "Albuquerque") // Filter by the nested field's full path
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const orders = await session.advanced
    .rawQuery("from Orders where ShipTo.City = $city")
    .addParameter("city", "Albuquerque")
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Orders
where ShipTo.City = "Albuquerque"
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/Orders/ByShipTo.City`  
> The auto-index is derived from the nested field path `ShipTo.City` referenced in the `where` clause.  
>
> The full dotted path is preserved in the auto-index name.
    
</Panel>

<Panel heading="Filter by values in a collection">
    
When the field you want to filter by is inside a collection, pass the full path to `whereEquals`.

For example, each _Order_ document contains a `Lines` collection.  
The following query returns every _Order_ document that contains at least one order line for  
`Teatime Chocolate Biscuits`.

<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const orders = await session
    .query({ collection: "Orders" })
    .whereEquals("Lines.ProductName", "Teatime Chocolate Biscuits")
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const orders = await session.advanced
    .rawQuery("from Orders where Lines[].ProductName = $productName")
    .addParameter("productName", "Teatime Chocolate Biscuits")
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Orders
where Lines[].ProductName = "Teatime Chocolate Biscuits"
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/Orders/ByLines[].ProductName`  
> The auto-index is derived from the array-element path `Lines[].ProductName` referenced in the `where` clause.
>    
> The `[]` notation in the auto-index name indicates that `Lines` is a collection and that the index is built over each element's `ProductName`.
    
</Panel>

<Panel heading="Filter by multiple possible values">

Use `whereIn` when a field can match any value from a specified list.      
For example, the following query returns _Employee_ documents whose first name is either `Robert` or `Nancy`. 
    
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const employees = await session
    .query({ collection: "Employees" })
    .whereIn("FirstName", ["Robert", "Nancy"])
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const employees = await session.advanced
    .rawQuery("from Employees where FirstName in ($names)")
    .addParameter("names", ["Robert", "Nancy"])
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Employees
where FirstName in ("Robert", "Nancy")
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/Employees/ByFirstName`  
> The auto-index is derived from the `FirstName` field referenced in the `where` clause.
>    
> The same auto-index serves both equality (`FirstName = "Robert"`) and `in` queries on this field -  
> the operator does not affect the auto-index name.  
> This means the auto-index created in [Filter by field value](#filter-by-field-value) is reused here,
  and no new auto-index is created.
    
</Panel>

<Panel heading="Filter by matching any value in a collection">
    
Use `containsAny` to return documents where a collection field contains at least one value from the specified list.  
    
Assume the following `BlogPost` class:    
 
<TabItem>
```javascript
class BlogPost {
    constructor() {
        this.id = null;
        this.title = null;
        this.tags = [];
    }
}
```    
</TabItem> 
    
The following query returns all blog posts that contain either the `Development` tag or the `Research` tag.  
The query will return a document if the `Tags` collection contains at least one of the specified values.
    
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const posts = await session
    .query({ collection: "BlogPosts" })
    .containsAny("tags", ["Development", "Research"])
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const posts = await session.advanced
    .rawQuery("from BlogPosts where tags in ($tags)")
    .addParameter("tags", ["Development", "Research"])
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from BlogPosts
where tags in ("Development", "Research")
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/BlogPosts/ByTags`  
> The auto-index is derived from the `Tags` collection field referenced in the `where` clause.
    
</Panel>

<Panel heading="Filter by matching all values in a collection">
    
Use `containsAll` to return documents where a collection field contains all values from the specified list.
For example, the following query returns only blog posts where the `Tags` collection contains both `Development` and `Research`.    
 
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const posts = await session
    .query({ collection: "BlogPosts" })
    .containsAll("tags", ["Development", "Research"])
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const posts = await session.advanced
    .rawQuery("from BlogPosts where tags all in ($tags)")
    .addParameter("tags", ["Development", "Research"])
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from BlogPosts
where tags all in ("Development", "Research")
```
</TabItem>
</Tabs>

> **Auto-index used:** `Auto/BlogPosts/ByTags`  
> The auto-index is derived from the `Tags` collection field referenced in the `where` clause.
>    
> `containsAny` (`in`) and `containsAll` (`all in`) produce the same auto-index name;  
> the operator only affects which documents match, not which auto-index is created or used.  
> This means the auto-index created for [Filter by matching any value in a collection](#filter-by-matching-any-value-in-a-collection) is reused here.
    
</Panel>

<Panel heading="Filter by document ID">
   
You can filter by document ID or by an ID prefix.
When the query only filters by the document ID, RavenDB does not need to create an auto-index.
Instead, the query is served directly by the storage engine.

This applies when the property used in the `whereEquals` clause is recognized as the identity property of the entity type,
according to the [`findIdentityProperty` convention](../../../client-api/configuration/identifier-generation/global.mdx#findidentityproperty),
and no other fields are involved in the query.

Such queries are considered [Collection queries](../../../querying/overview.mdx#3-query-a-collection---query-full-collection--query-by-id).
Simple collection queries that ask for documents by ID, or by an ID prefix, and do not require additional handling such as ordering or full-text search, have no additional indexing cost.

In terms of efficiency, querying by ID is the same as loading documents using [`session.load`](../../../client-api/session/loading-entities.mdx).

---
    
For example, the following query returns the order whose document ID is `orders/1-A`.    

<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const order = await session
    .query({ collection: "Orders" })
    .whereEquals("id", "orders/1-A")
    .firstOrNull();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const order = await session.advanced
    .rawQuery("from Orders where id() = $id")
    .addParameter("id", "orders/1-A")
    .firstOrNull();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Orders
where id() = "orders/1-A"
```
</TabItem>
</Tabs>

> **No auto-index is created**.  
> Filtering only by document ID is served directly by the storage engine.
    
---
    
You can also filter by an ID prefix to return all documents whose IDs start with a specified value:

<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">
```javascript
const orders = await session
    .query({ collection: "Orders" })
    .whereStartsWith("id", "orders/")
    .all();
```
</TabItem>
<TabItem value="RawQuery" label="RawQuery">
```javascript
const orders = await session.advanced
    .rawQuery("from Orders where startsWith(id(), $prefix)")
    .addParameter("prefix", "orders/")
    .all();
```
</TabItem>
<TabItem value="RQL" label="RQL">
```sql
from Orders
where startsWith(id(), "orders/")
```
</TabItem>
</Tabs>

> **No auto-index is created**.  
> Filtering by an ID prefix is also handled directly by the storage engine.
    
</Panel>

<Panel heading="Filtering when querying static indexes">

All filtering methods shown above can also be used when querying a **static index**.     
However, when querying a static index, the filter must reference fields that are defined by the index.    

For example, consider the following index:

<TabItem>
```javascript
class Employees_ByFirstAndLastName extends AbstractJavaScriptIndexCreationTask {
    constructor() {
        super();

        this.map("Employees", employee => {
            return {
                // The index fields
                FirstName: employee.FirstName,
                LastName:  employee.LastName
            };
        });
    }
}
```
</TabItem>

You can query the index and filter the resulting documents by the index fields.        
The following example queries the index for _Employee_ documents and filters them by the index field `FirstName`.
  
<Tabs groupId='languageSyntax'>
<TabItem value="Query" label="Query">

```javascript
const employees = await session
    .query({ index: Employees_ByFirstAndLastName })
    .whereEquals("FirstName", "Robert")
    .all();
```

</TabItem>
<TabItem value="RawQuery" label="RawQuery">

```javascript
const employees = await session.advanced
    .rawQuery(
        "from index 'Employees/ByFirstAndLastName' where FirstName = $firstName")
    .addParameter("firstName", "Robert")
    .all();
```

</TabItem>
<TabItem value="RQL" label="RQL">

```sql
from index "Employees/ByFirstAndLastName"
where FirstName = "Robert"
```

</TabItem>
</Tabs>    
    
</Panel>

<Panel heading="Syntax">

**Query filtering methods**

<TabItem>     
```typescript
// Equality and comparison
whereEquals(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;
whereNotEquals(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;

whereGreaterThan(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;
whereGreaterThanOrEqual(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;
whereLessThan(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;
whereLessThanOrEqual(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;

whereBetween(fieldName: string, start: any, end: any, exact?: boolean): IDocumentQuery<T>;

// Collection / set matching
whereIn(fieldName: string, values: any[], exact?: boolean): IDocumentQuery<T>;
containsAny(fieldName: string, values: any[]): IDocumentQuery<T>;
containsAll(fieldName: string, values: any[]): IDocumentQuery<T>;

// Prefix / suffix
whereStartsWith(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;
whereEndsWith(fieldName: string, value: any, exact?: boolean): IDocumentQuery<T>;

// Field presence
whereExists(fieldName: string): IDocumentQuery<T>;
```
</TabItem>
    
---    
                                                    
**`rawQuery`**

```typescript
// Submit an RQL string directly.
rawQuery<T>(query: string, documentType?: DocumentType<T>): IRawDocumentQuery<T>;

// Bind a value to an RQL parameter referenced as $paramName.
addParameter(name: string, value: any): IRawDocumentQuery<T>;
```

| Parameter      | Type      | Description                                                                                                            |
|----------------|-----------|------------------------------------------------------------------------------------------------------------------------|
| **fieldName**  | `string`  | The name of the field to filter on. Use a dotted path for nested fields<br/>(e.g. `"ShipTo.City"`, `"Lines.ProductName"`). |
| **value**      | `any`     | The value to compare against.                                                                                          |
| **values**     | `any[]`   | The set of values to compare against.                                                                                  |
| **exact**      | `boolean` | When `true`, string comparison is case-sensitive.<br/>Default: `false`.                                                |
| **query**      | `string`  | An RQL string.<br/>Use `$paramName` placeholders for values supplied via `addParameter`.                                   |
| **name**       | `string`  | The name of an RQL parameter (no `$` prefix).                                                                          |

</Panel>
