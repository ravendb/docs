<Admonition type="note" title="">

* The revision configuration rules are usually applied to a document's revisions only when the document is modified.  
  Use `EnforceRevisionsConfigurationOperation` to apply the current [revisions configuration](../../../../../studio/database/settings/document-revisions.mdx)
  to existing revisions, instead of waiting for each document to be modified.

* Running this operation scans existing revisions and enforces the current rules immediately,
  **deleting** eligible revisions that should be purged according to the configuration.

* The operation can be applied to all collections (default) or to a specified set of collections.

* By default, the operation is applied to the [default database](../../../../../client-api/setting-up-default-database.mdx).  
  To operate on a different database, see [switch operations to a different database](../../../../../client-api/operations/how-to/switch-operations-to-a-different-database.mdx).

* In this article:
  * [Enforce configuration on all collections](../../../../../document-extensions/revisions/client-api/operations/enforce-revisions-configuration.mdx#enforce-configuration-on-all-collections)
  * [Enforce configuration on specific collections](../../../../../document-extensions/revisions/client-api/operations/enforce-revisions-configuration.mdx#enforce-configuration-on-specific-collections)
  * [Include force-created revisions](../../../../../document-extensions/revisions/client-api/operations/enforce-revisions-configuration.mdx#include-force-created-revisions)
  * [Throttle the operation](../../../../../document-extensions/revisions/client-api/operations/enforce-revisions-configuration.mdx#throttle-the-operation)
  * [Syntax](../../../../../document-extensions/revisions/client-api/operations/enforce-revisions-configuration.mdx#syntax)

</Admonition>

<Admonition type="warning" title="Resource consumption">

* A large database may contain many revisions that exceed the current configuration limits.
  Running this operation can delete many revisions in a single operation and may require substantial server resources.
  Schedule it accordingly.

* On large datasets, use `setMaxOpsPerSecond(...)` to [throttle the operation](../../../../../document-extensions/revisions/client-api/operations/enforce-revisions-configuration.mdx#throttle-the-operation)
  and reduce its resource consumption.

* Revisions in collections that no current configuration applies to may be deleted.
  Make sure your revisions configuration includes the default settings and collection-specific configurations needed to keep the revisions you want to preserve.
  Force-created revisions may also be removed when `setIncludeForceCreated(true)` is used.

</Admonition>

<Panel heading="Enforce configuration on all collections">

In this example, the current revisions configuration is enforced on the revisions of all collections.

```java
// Define the enforce revisions configuration operation
EnforceRevisionsConfigurationOperation enforceConfigurationOp =
    new EnforceRevisionsConfigurationOperation();

// Execute the operation by passing it to operations().sendAsync
// 'sendAsync' returns an Operation object that can be awaited
Operation operation = store.operations().sendAsync(enforceConfigurationOp);

// Wait for the operation to complete
operation.waitForCompletion();
```

</Panel>

<Panel heading="Enforce configuration on specific collections">
    
Use `setCollections(...)` to enforce the configuration only on revisions in the specified collections.  
When `collections` is not set, is `null`, or is an empty array, the operation is applied to all collections.

```java
EnforceRevisionsConfigurationOperation.Parameters parameters =
    new EnforceRevisionsConfigurationOperation.Parameters();

// The configuration will be enforced only on these collections
parameters.setCollections(new String[]{ "Users", "Products" });

Operation operation = store.operations().sendAsync(
    new EnforceRevisionsConfigurationOperation(parameters));

operation.waitForCompletion();
```

</Panel>

<Panel heading="Include force-created revisions">

By default, [force-created revisions](../../../../../document-extensions/revisions/overview.mdx#force-revision-creation)
are preserved and are not removed by the operation.     
    
Call `setIncludeForceCreated(true)` to allow those force-created revisions to be removed as well.    

```java
EnforceRevisionsConfigurationOperation.Parameters parameters =
    new EnforceRevisionsConfigurationOperation.Parameters();

// Force-created revisions will also be subject to the configuration rules
parameters.setIncludeForceCreated(true);

Operation operation = store.operations().sendAsync(
    new EnforceRevisionsConfigurationOperation(parameters));

operation.waitForCompletion();
```

</Panel>

<Panel heading="Throttle the operation"> 

Running the operation on a large database may consume substantial server resources and affect server performance.    

Set `setMaxOpsPerSecond(...)` to limit the number of documents processed per second,  
throttling the operation to reduce its resource consumption.    

```java
EnforceRevisionsConfigurationOperation.Parameters parameters =
    new EnforceRevisionsConfigurationOperation.Parameters();

// Process at most 100 documents per second
parameters.setMaxOpsPerSecond(100);

Operation operation = store.operations().sendAsync(
    new EnforceRevisionsConfigurationOperation(parameters));

operation.waitForCompletion();
```

<Admonition type="warning" title="">

`maxOpsPerSecond` must be greater than `0`.  
Calling `setMaxOpsPerSecond` with `0` or a negative value throws an `IllegalStateException`.

</Admonition>

</Panel>

<Panel heading="Syntax">

```java
// Available constructors:
// ====================

// Enforce the configuration using default parameters (all collections)
public EnforceRevisionsConfigurationOperation();

// Enforce the configuration using the specified parameters
public EnforceRevisionsConfigurationOperation(Parameters parameters);
```

| Parameter        | Type         | Description                                                                                      |
|------------------|--------------|--------------------------------------------------------------------------------------------------|
| **parameters**   | `Parameters` | The parameters defining how the configuration is enforced.<br/>See the `Parameters` class below. |

```java
public static class Parameters {
    public boolean isIncludeForceCreated();
    public void setIncludeForceCreated(boolean includeForceCreated);

    public String[] getCollections();
    public void setCollections(String[] collections);

    public Integer getMaxOpsPerSecond();
    public void setMaxOpsPerSecond(Integer maxOpsPerSecond);
}
```
<br/>    

| Parameter               | Type       | Description |
|-------------------------|------------|-------------|
| **includeForceCreated** | `boolean`  | `true` - [Force-created revisions](../../../../../document-extensions/revisions/overview.mdx#force-revision-creation)may also be removed by the operation.<br/>`false` (default) - Force-created revisions are not included in deletion paths that explicitly protect them. |
| **collections**         | `String[]` | The collections the operation applies to.<br/>When not set or `null`, the operation applies to all collections. |
| **maxOpsPerSecond**     | `Integer`  | Limits the number of documents processed per second, throttling the operation to reduce resource consumption.<br/>When not set (`null`), no throttling is applied.<br/>Must be greater than `0`, otherwise an `IllegalStateException` is thrown. |

The operation returns an `OperationIdResult`.  
Use the returned `Operation` object to wait for the operation to complete via `waitForCompletion()`.
