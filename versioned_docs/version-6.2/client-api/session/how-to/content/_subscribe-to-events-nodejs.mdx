<Admonition type="note" title="">

* **Events** let you run custom logic in response to operations made on a `DocumentStore`
  or a `Session`.  
  An event is invoked when the selected action runs on an entity, or when a query is made.

* Subscribing to an event in a `Session` is valid for that session only.

* Subscribing to an event at the `DocumentStore` level subscribes to that event in all
  subsequent sessions.  
  Subscribe with `store.addSessionListener(eventName, handler)`, passing the event name,
  e.g. for the `afterSaveChanges` event:  
  `store.addSessionListener("afterSaveChanges", event => onAfterSaveChanges(event));`  
  Read more about store events [here](../../../how-to/subscribe-to-store-events.mdx).

### `beforeStore`

This event is invoked as a part of `saveChanges()`, but before it is actually sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: discontinue out-of-stock products

```js
function onBeforeStore(args) {
    if (args.getEntity() instanceof Product) {
        const product = args.getEntity();
        if (product.unitsInStock === 0) {
            product.discontinued = true;
        }
    }
}
```
<br />

After you subscribe to the event, every stored entity invokes the method:

```js
store.addSessionListener("beforeStore", event => onBeforeStore(event));

const session = store.openSession();
await session.store(new Product({ name: "RavenDB v3.5", unitsInStock: 0 }));
await session.store(new Product({ name: "RavenDB v4.0", unitsInStock: 1000 }));

await session.saveChanges(); // Here the method is invoked
```

</ContentFrame>

<ContentFrame>

### `beforeDelete`

This event is invoked by `delete(id)` or `delete(entity)`. It is executed only when `saveChanges()`
is called, but before the commands are actually sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: prevent deletions

```js
function onBeforeDelete(args) {
    throw new Error("Not implemented");
}
```
<br />

Subscribe to the event:

```js
store.addSessionListener("beforeDelete", event => onBeforeDelete(event));

const session = store.openSession();
const product = await session.load("products/1-A", Product);

// onBeforeDelete is triggered whether you
// call delete() on an entity or on its ID
await session.delete(product);
await session.saveChanges(); // Error will be thrown
```

</ContentFrame>

<ContentFrame>

### `afterSaveChanges`

This event is invoked after `saveChanges` returns.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: log each saved document

```js
function onAfterSaveChanges(args) {
    console.log("Document " + args.documentId + " was saved.");
}
```
<br />

Subscribe to the event:

```js
store.addSessionListener("afterSaveChanges", event => onAfterSaveChanges(event));
```

</ContentFrame>

<ContentFrame>

### `beforeQuery`

This event is invoked just before the query is sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: disable query caching

```js
function onBeforeQuery(args) {
    args.queryCustomization.noCaching();
}
```
<br />

#### Example: wait for non-stale results

To have each query [wait for non-stale results](../../../../indexes/stale-indexes.mdx),
define the event as follows:

```js
function onBeforeQuery(args) {
    args.queryCustomization.waitForNonStaleResults(30);
}
```
<br />

Subscribe to the event:

```js
store.addSessionListener("beforeQuery", event => onBeforeQuery(event));
```

</ContentFrame>

<ContentFrame>

### `beforeConversionToDocument`

This event is invoked before an entity is converted to a JSON document, e.g. when a document
is sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: modify an entity before conversion

```js
function onBeforeConversionToDocument(args) {
    if (args.entity instanceof Product) {
        const product = args.entity;
        product.before = true;
    }
}
```
<br />

Subscribe to the event:

```js
store.addSessionListener("beforeConversionToDocument", event => onBeforeConversionToDocument(event));
```

</ContentFrame>

<ContentFrame>

### `afterConversionToDocument`

This event is invoked after an entity is converted to a JSON document.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: act on a document after conversion

```js
function onAfterConversionToDocument(args) {
    if (args.entity instanceof Product) {
        const product = args.entity;
        product.after = true;
    }
}
```
<br />

Subscribe to the event:

```js
store.addSessionListener("afterConversionToDocument", event => onAfterConversionToDocument(event));
```

</ContentFrame>

<ContentFrame>

### `beforeConversionToEntity`

This event is invoked before a JSON document is converted to an entity, e.g. when a document
is loaded.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: modify a document before conversion

```js
store.addSessionListener("beforeConversionToEntity", (event) => {
    const document = ObjectUtil.clone(event.document);

    document.before = true;
    event.document = document;
});
```

</ContentFrame>

<ContentFrame>

### `afterConversionToEntity`

This event is invoked after a JSON document is converted to an entity.  
See its [arguments](#event-arguments) in the Syntax section below.

#### Example: modify an entity after conversion

```js
store.addSessionListener("afterConversionToEntity", (event) => {
    if (event.entity instanceof Item) {
        const item = event.entity;
        item.after = true;
    }
});
```

</ContentFrame>

</Panel>

<Panel heading="Unsubscribing from an event">

To stop responding to an event, detach the handler with `store.removeSessionListener`, passing
the same event name and handler you subscribed with:

```js
store.removeSessionListener("beforeStore", onBeforeStore);
```
<br />

Session events can be unsubscribed at any time.

</Panel>

<ContentFrame>

### Methods

```js
store.addSessionListener(eventName, eventHandler)
store.removeSessionListener(eventName, eventHandler)
```
<br />

`eventName` is the name of the session event to handle:

| Event name | Event arguments |
| --- | --- |
| `"beforeStore"` | `SessionBeforeStoreEventArgs` |
| `"beforeDelete"` | `SessionBeforeDeleteEventArgs` |
| `"afterSaveChanges"` | `SessionAfterSaveChangesEventArgs` |
| `"beforeQuery"` | `SessionBeforeQueryEventArgs` |
| `"beforeConversionToDocument"` | `BeforeConversionToDocumentEventArgs` |
| `"afterConversionToDocument"` | `AfterConversionToDocumentEventArgs` |
| `"beforeConversionToEntity"` | `BeforeConversionToEntityEventArgs` |
| `"afterConversionToEntity"` | `AfterConversionToEntityEventArgs` |

</ContentFrame>
