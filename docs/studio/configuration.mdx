# Studio configuration options
<Admonition type="note" title="">

Use the configuration keys listed below to set Studio-related options for your RavenDB server.  
Learn how to apply these keys in the [Configuration Overview](../server/configuration/configuration-options.mdx) article.

* On this page:  
   * [Configuration keys](../studio/configuration.mdx#configuration-keys)
      * [Studio.Path](../studio/configuration.mdx#studiopath)
      * [Ai.Assistant.Disable](../studio/configuration.mdx#aiassistantdisable)
      * [Ai.Assistant.DisableDataSubmission](../studio/configuration.mdx#aiassistantdisabledatasubmission)

</Admonition>

<Panel heading="Configuration keys">

<ContentFrame>

### Studio.Path

The directory in which RavenDB searches for Studio files.  
Defaults to the base directory of the RavenDB server installation.

- **Type**: `string`
- **Default**: `null` (base directory)
- **Scope**: Server-wide

</ContentFrame>

<ContentFrame>

### Ai.Assistant.Disable

Use this key to completely disable the [AI assistant](../studio/ai-assistant.mdx) for all users regardless of individual [user consent](../studio/ai-assistant.mdx#user-consent) and to block the `AI assistant` option in Studio's Knowledge Center panel.

- **Type**: `bool`
- **Default**: `false`
- **Scope**: Server-wide

</ContentFrame>

<ContentFrame>

### Ai.Assistant.DisableDataSubmission

Use this key to prevent the transmission of any user data [exposed to the AI assistant](../studio/ai-assistant.mdx#add-context) to the RavenDB backend or LLM. The assistant can still provide general guidance but cannot analyze any user data.

- **Type**: `bool`
- **Default**: `false`
- **Scope**: Server-wide

</ContentFrame>

</Panel>
