# GenAI: Start

RavenDB Generative AI (GenAI) tasks are ongoing operations that remain active in the background, continuously monitoring a specified collection and processing documents as they are added or modified.  
Similar to ETL tasks, a GenAI task extracts content from each processed document and transfers this content. Its destination, however, is an AI model, to which it sends the **content** along with a guiding **prompt** and a **response template**.  
When the model responds, its reply is structured in a way that allows the task to use it effectively. For example, the task can use the response to update the source document with the requested information.  
GenAI tasks can be used to summarize long documents, extract key details, translate texts, generate new content based on the existing data, and so on. They can be easily created using Studio or the client API, and when created via Studio, each step of their creation can be easily tested and validated before deployment.
