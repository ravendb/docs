# Cluster: Cluster API

<Admonition type="note" title="">

* The easiest way to manage a RavenDB cluster is through Studio's [Cluster View](../../studio/cluster/cluster-view.mdx).  
  All cluster operations are available through the GUI. However, sometimes you'd prefer to automate these operations.  

* In this section we will demonstrate how to perform certain cluster operations programmatically using the REST API.  
  The first example shows how to make a secure call using PowerShell, cURL, and the RavenDB C# client.  
  All other examples are cURL only.  

* If authentication is enabled, a client certificate with either `Cluster Admin` or `Operator` [security clearance](../../server/security/authorization/security-clearance-and-permissions.mdx) must be provided.  
  The security clearance each endpoint requires is noted below.  

* In this article:
   * [Add node to the cluster](../../server/clustering/cluster-api.mdx#add-node-to-the-cluster)
   * [Delete node from the cluster](../../server/clustering/cluster-api.mdx#delete-node-from-the-cluster)
   * [Promote a watcher node](../../server/clustering/cluster-api.mdx#promote-a-node)
   * [Demote a watcher node](../../server/clustering/cluster-api.mdx#demote-a-node)
   * [Force elections](../../server/clustering/cluster-api.mdx#force-elections)
   * [Force timeout](../../server/clustering/cluster-api.mdx#force-timeout)
   * [Bootstrap cluster](../../server/clustering/cluster-api.mdx#bootstrap-cluster)

</Admonition>

<Panel heading="Add node to the cluster">

You can add a node to the cluster using an `HTTP PUT` request to the `/admin/cluster/node` endpoint with the following arguments:

| Argument | Description | Required | Default |
| - | - | - | - |
| `new-node-url` | The address of the new node you want to add to the cluster | `true` | - |
| `new-node-tag` | 1-4 uppercase unicode letters | `false` | `A` - `Z` assigned by order of addition |
| `is-watcher` | Add the new node as a [watcher](../../server/clustering/rachis/cluster-topology.mdx#watcher) | `false` | `false` |
| `max-utilized-cores` | The maximum number of cores that can be assigned to the new node | `false` | Number of processors on the machine or the license limit, the smaller of the two |

* Security clearance: `Cluster Admin`

#### Example: Add a node

<Tabs groupId="languages">
<TabItem value="csharp" label="C#">

To make a secure call, the Document Store must be provided with the client certificate (see [example](../../client-api/setting-up-authentication-and-authorization.mdx#example---initializing-document-store-with-a-client-certificate)).

```csharp
store.GetRequestExecutor().HttpClient.SendAsync(new HttpRequestMessage(HttpMethod.Put, "https://<server-url>/admin/cluster/node?url=<new-node-url>&tag=<new-node-tag>&watcher=<is-watcher>&maxUtilizedCores=<max-utilized-cores>"));
```

</TabItem>
<TabItem value="powershell" label="PowerShell">

To make a secure call, load the client certificate from a PFX file and pass it to the request.

```powershell
[Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
$clientCert = Get-PfxCertificate -FilePath <path-to-pfx-cert>
Invoke-WebRequest -Method Put -URI "https://<server-url>/admin/cluster/node?url=<new-node-url>&tag=<new-node-tag>&watcher=<is-watcher>&maxUtilizedCores=<max-utilized-cores>" -Certificate $clientCert
```

</TabItem>
<TabItem value="curl" label="cURL">

To make a secure call, provide the client certificate and private key using the `--cert` and `--key` options.

```bash
curl --cert <path-to-cert> --key <path-to-key> -X PUT "https://<server-url>/admin/cluster/node?url=<new-node-url>&tag=<node-tag>&watcher=<is-watcher>&maxUtilizedCores=<max-utilized-cores>"
```

</TabItem>
</Tabs>

<Panel heading="Delete node from the cluster">

You can delete a node from the cluster using an `HTTP DELETE` request to the `/admin/cluster/node` endpoint with the following argument:

| Argument | Description | Required | Default |
| - | - | - | - |
| `node-tag` | The tag of the node to delete | `true` | - |

* Security clearance: `Cluster Admin`

#### Example: Delete a node

```bash
curl -X DELETE "https://<server-url>/admin/cluster/node?nodeTag=<node-tag>"
```

</Panel>

<Panel heading="Promote a node">

You can [promote a node](../../server/clustering/rachis/cluster-topology.mdx#nodes-states-and-types) using an `HTTP POST` request to the `/admin/cluster/promote` endpoint with the following argument:

| Argument | Description | Required | Default |
| - | - | - | - |
| `node-tag` | The tag of the node to promote | `true` | - |

* The POST request body should be empty.
* Security clearance: `Cluster Admin`

#### Example: Promote a node

```bash
curl -X POST "https://<server-url>/admin/cluster/promote?nodeTag=<node-tag>" -d ''
```

</Panel>

<Panel heading="Demote a node">

You can [demote a node](../../server/clustering/rachis/cluster-topology.mdx#nodes-states-and-types) using an `HTTP POST` request to the `/admin/cluster/demote` endpoint with the following argument:

| Argument | Description | Required | Default |
| - | - | - | - |
| `node-tag` | The tag of the node to demote | `true` | - |

* The POST request body should be empty.
* Security clearance: `Cluster Admin`

#### Example: Demote a node

```bash
curl -X POST "https://<server-url>/admin/cluster/demote?nodeTag=<node-tag>" -d ''
```

</Panel>

<Panel heading="Force elections">

You can force an election using an empty `HTTP POST` request to the `/admin/cluster/reelect` endpoint.

* Security clearance: `Operator`

#### Example: Force an election

```bash
curl -X POST "https://<server-url>/admin/cluster/reelect" -d ''
```

</Panel>

<Panel heading="Force timeout">

You can force a timeout using an empty `HTTP POST` request to the `/admin/cluster/timeout` endpoint.

* Security clearance: `Operator`

#### Example: Force a timeout

```bash
curl -X POST "https://<server-url>/admin/cluster/timeout" -d ''
```

</Panel>

<Panel heading="Bootstrap cluster">

You can bootstrap the cluster using an empty `HTTP POST` request to the `/admin/cluster/bootstrap` endpoint.  
Note that bootstrapping takes effect only while the server is in the [Passive state](../../server/clustering/rachis/cluster-topology.mdx#nodes-states-and-types). On a server that has already formed or joined a cluster, the request has no effect.

* Security clearance: `Cluster Admin`

#### Example: Bootstrap the cluster

```bash
curl -X POST "https://<server-url>/admin/cluster/bootstrap" -d ''
```

</Panel>
