# Generating embeddings: Start

[Embeddings](https://en.wikipedia.org/wiki/Embedding_(machine_learning)) are mathematical representations of content, where the content is converted into high-dimensional numerical vectors that capture semantic meanings and relationships.  
RavenDB allows you to create embeddings for document fields (e.g., texts, arrays, or metadata) and use these embeddings to enable semantic searches based on meaning rather than exact keyword matches.  
- To search over embeddings, you can use [Vector search](../vector-search/vector-search_start) operations. You can pre-generate the embeddings for your documents using ongoing embeddings-generation tasks, or generate the embeddings on-the-fly during the search operation.  
- When you run a vector search, your search query is also converted into an embedding and compared against the document embeddings.  
- Both **embeddings-generation tasks** and **vector search** can create embeddings using an external AI provider (such as OpenAI) or RavenDB's default embedding model. Make sure you use the same model to both generate and query the embeddings.  
- Vector search and the generated embeddings can be used by other RavenDB AI features, including GenAI tasks and AI agents.
