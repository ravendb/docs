#Miscellaneous conventions



Disable all [profiling support](../../../how-to/enable-profiling.mdx) which has been enabled by the `DocumentStore.initializeProfiling()` call.

<TabItem value="disable_profiling" label="disable_profiling">
<CodeBlock language="java">
{`conventions.setDisableProfiling(true);
`}
</CodeBlock>
</TabItem>

### MaxNumberOfRequestsPerSession

The max number of requests per session. See the [related article](../../../session/configuration/how-to-change-maximum-number-of-requests-per-session.mdx) for details.

<TabItem value="max_number_of_requests_per_session" label="max_number_of_requests_per_session">
<CodeBlock language="java">
{`conventions.setMaxNumberOfRequestsPerSession(10);
`}
</CodeBlock>
</TabItem>

### SaveEnumsAsIntegers

It determines if java `enum` types should be saved as integers or strings and instruct the query provider to query enums as integer values. Default: `false`.

<TabItem value="save_enums_as_integers" label="save_enums_as_integers">
<CodeBlock language="java">
{`conventions.setSaveEnumsAsIntegers(false);
`}
</CodeBlock>
</TabItem>

### DefaultUseOptimisticConcurrency

This convention allows to enable optimistic concurrency for all opened sessions. More about optimistic concurrency you will find [here](../../../session/configuration/how-to-enable-optimistic-concurrency.mdx).
By default concurrency checks are turned off:

<TabItem value="use_optimistic_concurrency_by_default" label="use_optimistic_concurrency_by_default">
<CodeBlock language="java">
{`conventions.setDefaultUseOptimisticConcurrency(false);
`}
</CodeBlock>
</TabItem>
