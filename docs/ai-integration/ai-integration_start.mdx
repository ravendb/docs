# AI Integration: Start

RavenDB is equipped with a set of powerful native AI features that can 
be used independently or in conjunction with each other, allowing you to easily integrate advanced AI capabilities into your applications.  
These features include [Embeddings generation](../ai-integration/ai-integration_start#embeddings-generation), [Vector search](../ai-integration/ai-integration_start#vector-search), [AI agents](../ai-integration/ai-integration_start#ai-agents), and [GenAI tasks](../ai-integration/ai-integration_start#genai-tasks).  

### Embeddings generation
[Embeddings](https://en.wikipedia.org/wiki/Embedding_(machine_learning)) are mathematical representations of content, where the content is converted into high-dimensional numerical vectors that capture semantic meanings and relationships.  
- RavenDB allows you to create embeddings for document fields and attachments, and apply **vector search operations** over the embeddings to find content by meaning or context rather than by exact keyword matches.  
- Embeddings can be created and stored by native RavenDB [embeddings-generation tasks](../ai-integration/generating-embeddings/embeddings-generation_start) that systematically process document collections and convert document fields (like texts or arrays) into embeddings. Such tasks can use either an external AI model (such as OpenAI) or RavenDB's default embedding model.  
- You can also store in the database embeddings created by external embeddings providers (e.g., embeddings created for images), or avoid pre-generating embeddings and let vector search operations generate them on-the-fly, while searching.

### Vector search
[Vector search](../ai-integration/vector-search/vector-search_start) operations let you find related content by comparing embeddings, e.g. to find texts by meaning or images by context.  
- When running a vector search, your query is converted into an embedding and compared against document embeddings. The document embeddings can be either pre-generated (by an embeddings-generation task or an external provider) or generated on-the-fly while searching.  
- You can run vector search using either a dynamic query for ad-hoc or infrequent searches or a static index for optimized performance.  
- Vector search is designed so other RavenDB AI features can easily use it. A [GenAI task](../ai-integration/gen-ai-integration/gen-ai_start), for example,  can use vector search to find documents and update them with LLM-generated content, and an [AI agent](../ai-integration/ai-agents/ai-agents_start) can use vector search to retrieve relevant data requested by the LLM.

### GenAI tasks
[GenAI tasks](../ai-integration/gen-ai-integration/gen-ai_start) are [ongoing operations](../studio/database/tasks/ongoing-tasks/general-info) that continuously monitor specified collections, processing documents as they are added or modified.  
- Similar to ETL tasks, a GenAI task extracts content from documents. But instead of sending the content to another database, the task sends it to an AI model (like OpenAI) along with a guiding **prompt** and a **JSON schema** that defines the layout for the model's response.  
- When the LLM responds, the GenAI task can use its response to, for example, update the source document, or create new documents in the database.  
- GenAI tasks can infuse intelligence into a wide variety of content handling scenarios, like enriching documents with AI-generated summaries or classifications, translating text into different languages, or generating new content based on existing data.

### AI agents
[AI agents](../ai-integration/ai-agents/ai-agents_start) are built to easily integrate AI capabilities into RavenDB clients.  
- An AI agent is a component that resides on the server and serves as a client's proxy to an AI model; the agent can maintain a continuous conversation with the model, and enable the model to securely query a RavenDB database and request the client to perform actions.  
- Using AI agents frees developers from the need to manage an AI model in their code, allows an almost immediate integration of numerous operational scenarios in your clients, and enhances the AI model's performance by giving it access to a credible and relevant data source.
