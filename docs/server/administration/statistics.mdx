# Administration: Statistics

<Admonition type="note" title="">

* RavenDB reports runtime statistics and metrics for the server and for individual databases.  

* You can view this data in Studio, or retrieve it in JSON format from dedicated endpoints.  

* RavenDB can also be monitored via protocols like [SNMP](../../monitoring/integrations/snmp/overview.mdx) and
  [OTLP](../../monitoring/integrations/open-telemetry.mdx) (used by OpenTelemetry), as well as monitoring tools like [Prometheus](../../monitoring/integrations/prometheus.mdx).

## Server statistics via Studio

Find server statistics in the following Studio views:

* [Cluster Dashboard](../../studio/cluster/cluster-dashboard/cluster-dashboard-overview.mdx): live per-node CPU, memory, traffic, indexing, and storage metrics.
* [Cluster View](../../studio/cluster/cluster-view.mdx#cluster-view-stats): cluster-wide stats, including assigned and licensed cores, election term, and per-node state and memory.

</ContentFrame>

## Metrics

<Admonition type="note" title="">
Get server metrics using the `{serverUrl}/admin/metrics` endpoint.
</Admonition>

A document containing the following metrics is retrieved:

* **Requests**
    * RequestsPerSec
    * ConcurrentRequestsCount
* **Docs**
    * BytesPutsPerSec
    * PutsPerSec
* **Attachments**
    * BytesPutsPerSec
    * PutsPerSec
* **MapIndexes**
    * IndexedPerSec
* **MapReduceIndexes**
    * MappedPerSec
    * ReducedPerSec

Each meter has the following format.

```json
{
    Current: 3,
    Count: 217937,
    MeanRate: 0.2,
    OneMinuteRate: 1.2,
    FiveMinuteRate: 0.2,
    FifteenMinuteRate: 0.1
}
```

</ContentFrame>

## Database statistics via Studio

Several Studio views show database statistics:

* [Database List View](../../studio/database/databases-list-view.mdx#database-stats): general database stats, including size, document and index counts, and status.
* [Indexing Performance](../../studio/database/indexes/indexing-performance.mdx#index-statistics-view): per-index statistics, including map and reduce attempts, success counts, and rates.
* [Storage Report](../../studio/database/stats/storage-report.mdx): storage usage per collection and index.
