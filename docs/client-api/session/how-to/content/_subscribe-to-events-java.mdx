<Admonition type="note" title="">

* **Events** let you run custom logic in response to operations made on a `DocumentStore`
  or a `Session`.  
  An event is invoked when the selected action runs on an entity, or when a query is made.

* Subscribing to an event in a `Session` is valid for that session only.

* Subscribing to an event at the `DocumentStore` level subscribes to that event in all
  subsequent sessions.  
  Subscribe with the matching `add*Listener` method on the store, e.g. for the
  `AfterSaveChanges` event:  
  `store.addAfterSaveChangesListener(this::onAfterSaveChangesEvent);`  
  Read more about store events [here](../../../../client-api/how-to/subscribe-to-store-events.mdx).

### `beforeStore`

This event is invoked as a part of `saveChanges`, but before it is actually sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

```java
private void onBeforeStoreEvent(Object sender, BeforeStoreEventArgs args);
```
<br />

#### Example: discontinue out-of-stock products

```java
private void onBeforeStoreEvent(Object sender, BeforeStoreEventArgs args) {
    if (args.getEntity() instanceof Product) {
        Product product = (Product) args.getEntity();
        if (product.getUnitsInStock() == 0) {
            product.setDiscontinued(true);
        }
    }
}
```
<br />

After you subscribe to the event, every stored entity invokes the method:

```java
// subscribe to the event
store.addBeforeStoreListener(this::onBeforeStoreEvent);

try (IDocumentSession session = store.openSession()) {
    Product product1 = new Product();
    product1.setName("RavenDB v3.5");
    product1.setUnitsInStock(0);
    session.store(product1);

    Product product2 = new Product();
    product2.setName("RavenDB v4.0");
    product2.setUnitsInStock(1000);
    session.store(product2);

    session.saveChanges();  // Here the method is invoked
}
```

### `beforeDelete`

This event is invoked by `delete(id)` or `delete(entity)`. It is executed only when `saveChanges()`
is called, but before the commands are actually sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

```java
private void onBeforeDeleteEvent(Object sender, BeforeDeleteEventArgs args);
```
<br />

#### Example: prevent deletions

```java
private void onBeforeDeleteEvent(Object sender, BeforeDeleteEventArgs args) {
    throw new NotImplementedException("Sample");
}
```
<br />

Subscribe to the event:

```java
// subscribe to the event
store.addBeforeDeleteListener(this::onBeforeDeleteEvent);

// open a session and delete an entity
try (IDocumentSession session = store.openSession()) {
    Product product = session.load(Product.class, "products/1-A");

    session.delete(product);
    session.saveChanges(); // NotImplementedException will be thrown here
}
```

</ContentFrame>

### `afterSaveChanges`

This event is invoked after `saveChanges` returns.  
See its [arguments](#event-arguments) in the Syntax section below.

```java
private void onAfterSaveChangesEvent(Object sender, AfterSaveChangesEventArgs args);
```
<br />

#### Example: log each saved document

```java
private void onAfterSaveChangesEvent(Object sender, AfterSaveChangesEventArgs args) {
    if (log.isLoggable(Level.INFO)) {
        log.info("Document " + args.getDocumentId() + " was saved");
    }
}
```
<br />

Subscribe to the event:

```java
store.addAfterSaveChangesListener(this::onAfterSaveChangesEvent);
```

</ContentFrame>

<ContentFrame>

### `beforeQuery`

This event is invoked just before the query is sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

```java
private void onBeforeQueryEvent(Object sender, BeforeQueryEventArgs args);
```
<br />

#### Example: disable query caching

```java
private void onBeforeQueryEvent(Object sender, BeforeQueryEventArgs args) {
    args.getQueryCustomization().noCaching();
}
```
<br />

#### Example: wait for non-stale results

To have each query [wait for non-stale results](../../../../indexes/stale-indexes.mdx),
define the event as follows:

```java
private void onBeforeQueryEvent(BeforeQueryEventArgs args) {
    args.getQueryCustomization().waitForNonStaleResults(Duration.ofSeconds(30));
}
```
<br />

Subscribe to the event:

```java
store.addBeforeQueryListener(this::onBeforeQueryEvent);
```

</ContentFrame>

### `beforeConversionToDocument`

This event is invoked before an entity is converted to a JSON document, e.g. when a document
is sent to the server.  
See its [arguments](#event-arguments) in the Syntax section below.

```java
private void onBeforeConversionToDocument(Object sender, BeforeConversionToDocumentEventArgs args);
```
<br />

#### Example: modify an entity before conversion

```java
private void onBeforeConversionToDocument(Object sender, BeforeConversionToDocumentEventArgs args) {
    if (args.getEntity() instanceof Item) {
        Item item = (Item) args.getEntity();
        item.setBefore(true);
    }
}
```
<br />

Subscribe to the event:

```java
store.addBeforeConversionToDocumentListener(this::onBeforeConversionToDocument);
```

</ContentFrame>

### `beforeConversionToEntity`

This event is invoked before a JSON document is converted to an entity, e.g. when a document
is loaded.  
See its [arguments](#event-arguments) in the Syntax section below.


<br />

#### Example: log a loaded document before conversion

```java
private void onBeforeConversionToEntity(Object sender, BeforeConversionToEntityEventArgs args) {
    if (args.getId() == "item/1-A") {
        if (log.isLoggable(Level.INFO)) {
            log.info("Document " + args.getId() + " has found");
        }
    }
}
```

Subscribe to the event:


<Panel heading="Unsubscribing from an event">

To stop responding to an event, detach the handler with the matching `remove*Listener` method,
passing the same handler you subscribed with:

```java
store.removeBeforeStoreListener(this::onBeforeStoreEvent);
```
<br />

Session events can be unsubscribed at any time.

</Panel>
