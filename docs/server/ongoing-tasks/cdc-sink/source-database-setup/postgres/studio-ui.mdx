# CDC Sink for PostgreSQL: Studio UI

<Admonition type="note" title="">

* CDC Sink tasks are created and managed through the **Ongoing Tasks** section of the
  Management Studio.

* In this article:
  * [Creating a CDC Sink Task](#creating-a-cdc-sink-task)
  * [Monitoring Task State](#monitoring-task-state)
  * [Editing a Task](#editing-a-task)

</Admonition>

<Panel heading="Creating a CDC Sink Task">

To create a CDC Sink task in the Studio:

1. Navigate to **Databases** → your database → **Ongoing Tasks**
2. Click **Add Task** and select **CDC Sink**
3. Configure the connection string pointing to the PostgreSQL source
4. Add one or more root tables with their column mappings
5. (Optional) Add embedded tables and linked tables for each root table
6. (Optional) Configure patches for INSERT/UPDATE/DELETE events
7. Click **Save**

</Panel>

<Panel heading="Monitoring Task State">

The task list shows:

* **Task name** and **connection string name**
* **Task state** and **connection status** - see
  [Ongoing Tasks - General Info](../../../../../studio/database/tasks/ongoing-tasks/general-info.mdx)
  for the full set of state and connection-status values
* **Responsible node** - the cluster node currently running the task
* **Progress** - during initial load, shows which tables have been scanned

Clicking on a task opens the detail view with per-table statistics.

</Panel>

<Panel heading="Editing a Task">

To edit a task, click its name in the Ongoing Tasks list. The same configuration
form used for creation opens in edit mode.

<Admonition type="warning" title="">
The replication slot and publication names are **fixed at task creation** and do not
change when you edit the task - regardless of whether names were auto-generated or
specified explicitly.

When you add or remove tables, CDC Sink continues using the same slot and publication
names. The **publication** in PostgreSQL controls which tables are replicated and must
be kept in sync with the task configuration:

* **Adding a table:** If CDC Sink has the necessary permissions, it will automatically
  run `ALTER PUBLICATION <name> ADD TABLE <table>;` when the task is saved. If it does
  not have permissions, a database administrator must run this manually.
* **Removing a table:** `ALTER PUBLICATION <name> DROP TABLE <table>;` must be run
  manually by a database administrator - CDC Sink does not remove tables from the
  publication automatically.

Orphaned slots and publications must be dropped manually by the database administrator.
See [Cleanup and Maintenance](./cleanup-and-maintenance.mdx).
</Admonition>

</Panel>
